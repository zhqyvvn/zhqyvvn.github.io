Study Notes of JUnit Testing (1)

Posted on 2022-02-27 Edited on 2023-02-07 In Study Notes , JUnit Views:

This post records some utilities of JUnit 4, which is legacy now.
For the latest JUnit version and its changes, check the offical document Junit 5

1. Assert

JUnit provides overloaded assertion methods.
Parameter order is (optional message, expected value, actual value).
The String type failure message is optional to be added as the first parameter.
Value types that can be compared: Arrays, single values, Objects, null values.

2. Suite

Run tests in different test classes at the same time.

Explanation for these annotations from the JUnit 4 offical doc:

  • @BeforeClass
  • @AfterClass
  • @Test

3. Exception Handling

The @Test annotation supports two optional parameters.
The first, expected, declares that a test method should throw an exception. If it doesn’t throw an exception or if it throws a different exception than the one declared, the test fails.

5. Parameterized Tests

In the above example, each instance of FibonacciTest will be constructed using the two-argument constructor and the data values in the @Parameters method.

In order to easily identify the individual tests, we can provide a name for the @Parameters annotation.
This name is allowed to contain placeholders, which are replaced at runtime. The placeholders are:

  • {index}: the current parameter index
  • {0}: the first parameter value
  • {1}: the second parameter value

In the example given above, the Parameterized runner creates names like test[3: fib(3)=2]. test is the method name of the @Test.
