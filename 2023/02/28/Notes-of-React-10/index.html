Study Notes of React (10) - The State Hook

Posted on 2023-02-28 Edited on 2023-03-14 In Study Notes , React Views: Word count in article: 1.2k

React Hooks are functions that manage the internal state of components and handle post-rendering side effects directly from the function components.

Hooks don’t work inside classes.

Function components and React Hooks do not replace class components. They are completely optional.

Hooks let us “hook into” state and lifecycle features directly from the function components.

React offers a number of built-in Hooks. A few of these include useState(), useEffect(), useContext(), useReducer(), and useRef(). See the full list in the docs.

setToggle() is called with the new state value as an argument, to update the value of toggle and re-render the component with the new value.

With the State Hook, state is updated by calling a state setter function, which is different from “binding functions to class instances, working with constructors, or dealing with the this keyword” in the class components.

Calling the state setter signals to React that the component needs to re-render, so the whole function defining the component is called again.

useState() allows React to keep track of the current value of state from one render to the next.

2. Initialize State

To initialize the state with a value, we can pass the initial value as an argument to the useState() function call.

There are three ways in which this code affects our component:

  1. During the first render, the initial state argument is used.
  2. When the state setter is called, React ignores the initial state argument and uses the new value.
  3. When the component re-renders for any other reason, React continues to use the same value from the previous render.

If we don’t pass an initial value when calling useState(), then the current value of the state during the first render will be undefined.

If we don’t have the value needed during the first render, we can explicitly pass null instead of just passively leaving the value as undefined.
