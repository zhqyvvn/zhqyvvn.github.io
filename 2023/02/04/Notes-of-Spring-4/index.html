Study Notes of Spring (4) - Spring Boot

Posted on 2023-02-04 Edited on 2023-02-05 In Study Notes , Spring Views:

Spring Boot is a collection of tools that extends the Spring framework and makes it easier to build applications quickly.

When the app is run, this controller class will be registered as a bean and used to define routes available in the API.

The app is doing a lot more than running the controller code:

  • A Spring Data JPA repository is bootstrapped
  • A Tomcat server is initialized
  • A WebApplicationContext is initialized
  • An H2Dialect is used
  • An option to Explicitly configure spring.jpa.open-in-view
  • The Tomcat server is started on port(s): 4000

1. “Starters”

In the pom.xml of the Spring Boot application, see the spring-boot-start-jpa and spring-boot-starter-web dependencies:

These are called Spring Boot “starters”. Each one represents a set of dependencies with which to run the application.

The spring-boot-starter-web dependency instructs Spring Boot to configure Tomcat for running a server, and configure Spring MVC (Model-View-Controller) for routing HTTP requests.

spring-boot-starter-data-jpa sets up JPA and Hibernate for database access.

There are other starters like spring-boot-starter-test which includes common testing libraries such as JUnit Jupiter, Hamcrest, and Mockito; and spring-boot-starter-parent, which provides the spring-boot:run command.

2. Auto-Configuration

See the SpringApplication class and @SpringBootApplication annotation from the org.springframework.boot package:

SpringApplication

The SpringApplication class is responsible for starting up the IoC container:

In Java, the IoC container is represented by an application context. It receives beans and enables Spring to inject them into other components.

In Java, the application context appears as a ApplicationContext interface. In the above line, the app is using the class WebApplicationContext, a web-specific implementation of ApplicationContext.

@SpringBootApplication

@Configuration and @ComponentScan are from the Spring framework. @EnableAutoConfiguration is from Spring Boot.

@EnableAutoConfiguration … tells Spring Boot to “guess” how you want to configure Spring, based on the jar dependencies that you have added. Since spring-boot-starter-web added Tomcat and Spring MVC, the auto-configuration assumes that you are developing a web application and sets up Spring accordingly.

Spring Boot knows to bootstrap JPA repositories, start a Tomcat server, and use an H2 dialect because spring-boot-starter-jpa, spring-boot-starter-web, and h2 are listed as dependencies in the pom.xml.
