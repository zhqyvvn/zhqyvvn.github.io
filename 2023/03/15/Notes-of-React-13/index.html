2. A Style Object Variable

If we want to use more than just a few styles, that’s often nicer is to store a style object in a variable, and then inject that variable into JSX.

For example,

Defining a variable named styles in the top-level scope would be an bad idea in many JavaScript environments. However, it’s totally fine in React. Since every file is invisible to every other file, except for what we choose to expose via module.exports, we could have hundreds of different files, all with global variables named styles, and there could be no conflicts.

Here we aren’t injecting an object literal, so we don’t need the double curly braces.

4. Style Value Syntax

In regular JS, style values are almost always strings. Even if a style value is numeric, we usually have to write it as a string so that we can specify a unit. For example, "450px" or "20%".
