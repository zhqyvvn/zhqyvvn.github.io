Study Notes of React (12) - Stateful and Stateless Components

Posted on 2023-03-14 In Study Notes , React Views: Word count in article: 698

“Stateful” describes any component that has a state property, whereas “stateless” describes any component that does not.

1. Pass state from Stateful Components to Stateless Components

In this pattern, a stateful component passes its state down to a stateless component.

In the Parent.js, Parent is the stateful component class. Since Parent is stateful, it needs to set its initial state with a constructor method.

In the Child.js, Child is the stateless component class.

props and state store dynamic information. Dynamic information can change.

  • A React component should use props to store information that can only be changed by a different component. A component should never update its this.props.
  • A React component should use state to store information that the component itself can change.

2. Child Components update the state of Parent Components

When the parent component passes a function with setState() inside to the child component, the child component can change the parent component’s state by calling that function.
