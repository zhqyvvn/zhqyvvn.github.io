Study Notes of React (16) - Input of the React Form

Posted on 2023-03-16 Edited on 2023-03-17 In Study Notes , React Views: Word count in article: 534

With the input in a React form, we want the server to know about every new character or deletion, as soon as it happens. That way, the screen will always be in sync with the rest of the application.

Here is an example of a basic input component:

In the above example, <input /> has an onChange attribute and it listens to a “change” event. When a user types or deletes in the <input />, that triggers the change event.

The change event is handled by the handleUserInput function. Inside handleUserInput, this.setState is called to set the userInput state equal to e.target.value. Since e.target.value equals to the text in the <input /> field, we set this.state.userInput equal to whatever text is currently in <input />.

Any time that a user updates <input />, the new text is immediately stored in the state of this Input component. We could easily connect that state with a server, and the text could be sent somewhere to be stored on every character change.

To make the text in <input /> responsive to this.state.userInput, we set its value attribute to value={this.state.userInput}. This makes <input /> changed from an uncontrolled component to a controlled component.

In React, without the value attribute, the <input /> keeps track of its own information and maintains its own internal state. After giving <input /> the value attribute, it stops using its internal storage and has no memory. It gets information of itself through props - here, the value.

The uncontrolled component maintains its own internal state, and the controlled component does not. Since the controlled component has no state, it must be controlled by something else. Most React components are controlled.

More information about controlled and uncontrolled components in the React Forms documentation.

Side Notes

Regarding the question: “Is e.target.value and the value of <input value=”” /> the same thing?”, here is some personal thoughts:

e.target.value and the value of <input value=”” /> are different things.

The first one reflects the input in the text box. The second one is an attribute of the JSX.

In React, the value attribute makes <input /> a controlled component, and forces it to have what’s in value as the inner text.

When the user edits in the input box, e.target.value changes, then the event handler sets new state according to the new e.target.value, and the <input> value attribute changes according to the new state. After that, the value attribute backfills the inner text of the <input />, and makes input.value (e.target.value) have the same value (it already has and won’t be changed thus that).

Quotes from Controlling an input with a state variable:

React will force the input to always have the value you passed.

When you control an input by passing some value to it, you force it to always have the value you passed.

Btw, in HTML, the value attribute is the default intial value of the input field.
